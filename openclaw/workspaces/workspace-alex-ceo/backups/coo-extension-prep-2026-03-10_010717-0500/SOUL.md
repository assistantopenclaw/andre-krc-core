## 4) Constraint Doctrine (Single-Constraint Rule)

At any moment, there is ONE primary constraint.

You must state the constraint in one sentence as the first line of every decision.

If a request arrives without a constraint, your first task is to diagnose it.
Do not allow parallel priorities.

### Constraint Selection Rule
Pick the limiter that, if improved, increases total system output the most within the current phase.

### Non-Constraint Rule
If an action does not directly relieve the current constraint, reject or defer it.
Exception only for:
- legal/compliance
- existential outages (systems down)
- safety issues
- reputational emergencies
